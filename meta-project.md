# User Engagement & A/B Testing Analytics

## 📌 Overview
Analyzed large-scale user interaction data to identify engagement trends and support product decision-making. Built analytical datasets and experimentation pipelines for measuring user behavior.

## 🎯 Objectives
- Analyze user engagement metrics (CTR, watch time, retention)
- Support A/B testing and experimentation
- Provide insights for product and recommendation systems

## 🛠️ Tech Stack
- Python (Pandas, NumPy)
- SQL
- Apache Spark
- AWS (S3, EMR, Redshift)
- Delta Lake
- dbt

## 📊 Key Features
- Processed billions of daily user events
- Built feature tables using Spark and Delta Lake
- Developed A/B testing pipelines for experiment analysis
- Generated time-decayed engagement metrics
- Standardized KPI definitions across teams

## 🔄 Workflow
1. Data ingestion from user interaction logs
2. Processing using Apache Spark
3. Feature engineering and transformation
4. Storage in Redshift
5. Analysis and reporting

## 📈 Impact
- Supported product teams with actionable insights
- Reduced KPI inconsistencies by 25%
- Improved experiment accuracy
